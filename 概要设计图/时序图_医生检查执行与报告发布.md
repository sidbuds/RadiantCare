# 时序图-医生检查执行与报告发布

```
sequenceDiagram
    participant D as 医生
    participant F as 前端
    participant B as 后端
    participant DB as 数据库
    D->>F: 查看待检任务
    F->>B: GET /doctor/exam-tasks
    B->>DB: 查询任务列表
    DB-->>B: 返回数据
    B-->>F: 返回任务列表
    F-->>D: 显示待检任务
    D->>F: 开始检查
    F->>B: POST /items/start
    B->>DB: 更新状态
    DB-->>B: 成功
    B-->>F: 成功
    D->>F: 录入检查结果
    F->>B: POST /items/results
    B->>DB: 保存结果
    DB-->>B: 成功
    B-->>F: 成功
    D->>F: 完成检查
    F->>B: POST /items/complete
    B->>DB: 更新状态
    DB-->>B: 成功
    B-->>F: 成功
    D->>F: 生成报告
    F->>B: POST /reports/generate
    B->>DB: 归集结果生成报告
    DB-->>B: 成功
    D->>F: 审核报告
    F->>B: POST /reports/review
    B->>DB: 更新审核状态
    D->>F: 发布报告
    F->>B: POST /reports/publish
    B->>DB: 更新发布状态
    DB-->>B: 成功
    B-->>F: 发布成功
```
