# 时序图-咨询管理流程

```
sequenceDiagram
    participant U as 用户
    participant A as 管理员
    participant D as 医生
    participant F as 前端
    participant B as 后端
    participant DB as 数据库
    U->>F: 发起咨询
    F->>B: POST /consultations
    B->>DB: 创建咨询记录
    DB-->>B: 成功
    B-->>F: 返回咨询编号
    F-->>U: 咨询提交成功
    A->>F: 查看待分配咨询
    F->>B: GET /admin/consultations
    B->>DB: 查询列表
    DB-->>B: 返回数据
    A->>F: 分配医生
    F->>B: POST /consultations/assign
    B->>DB: 更新分配
    DB-->>B: 成功
    D->>F: 查看待回复咨询
    F->>B: GET /doctor/consultations
    B->>DB: 查询列表
    DB-->>B: 返回数据
    D->>F: 输入回复
    F->>B: POST /consultations/reply
    B->>DB: 保存回复
    DB-->>B: 成功
    B-->>F: 回复成功
    U->>F: 查看咨询回复
    F->>B: GET /consultations/mine
    B->>DB: 查询回复
    DB-->>B: 返回数据
    B-->>F: 返回回复内容
    F-->>U: 显示医生回复
```
