# 熙心健康体检平台-接口需求文档

## 1. 文档范围
本文档用于收敛当前项目**真实已实现的接口能力**，作为前后端联调、后续扩能和回归验证的接口需求依据。内容来源为当前仓库后端 Controller、DTO 与现有路由实现。

## 2. 接口总览
当前接口按业务域拆分如下：
- 认证与用户信息：`/api/auth/*`
- 公开浏览接口：`/api/public/*`
- 用户预约接口：`/api/appointments/*`
- 用户订单接口：`/api/orders/*`
- 用户体检任务接口：`/api/exam-tasks/*`
- 用户报告接口：`/api/reports/*`
- 历年对比与建议接口：`/api/report-compare/*`、`/api/health-advices/*`
- 用户咨询接口：`/api/doctor-consultations/*`
- 医生检查接口：`/api/doctor/exam-tasks/*`
- 医生报告接口：`/api/doctor/reports/*`
- 医生咨询接口：`/api/doctor/consultations/*`
- 医生分析接口：`/api/doctor/analytics/*`
- 运营管理接口：`/api/operator/*`
- 管理端接口：`/api/admin/*`
- 支付回调接口：`/api/payment/*`

## 3. 通用约束
### 3.1 鉴权
- 需要登录的接口统一使用 `Authorization: Bearer <token>`。
- JWT 登录成功后返回：`accessToken`、`tokenType`、`expiresIn`、`role`、`accountId`、`userId`、`displayName`。
- 公开接口不需要登录。

### 3.2 角色约束
- `USER`：普通用户接口。
- `DOCTOR`：医生/医技接口。
- `OPERATOR`：运营接口。
- `ADMIN`：管理员接口。
- 公开接口：无角色要求。

### 3.3 返回结构
统一返回结构为：
```json
{
  "code": 0,
  "message": "...",
  "data": "..."
}
```

## 4. 认证接口
### 4.1 注册
- 方法：`POST`
- 路径：`/api/auth/register`
- 权限：公开
- 请求字段：
  - `username`：必填，4~50 位，只允许字母数字下划线
  - `password`：必填，6~128 位
  - `name`：必填，最多 50 位
  - `mobile`：必填，符合 `^1[3-9]\\d{9}$`
- 返回字段：
  - `accessToken`
  - `tokenType`
  - `expiresIn`
  - `role`
  - `accountId`
  - `userId`
  - `displayName`

### 4.2 登录
- 方法：`POST`
- 路径：`/api/auth/login`
- 权限：公开
- 请求字段：
  - `username`：必填
  - `password`：必填
- 返回字段：同注册接口

### 4.3 当前用户信息
- 方法：`GET`
- 路径：`/api/auth/me`
- 权限：已登录
- 返回：当前登录用户上下文信息

## 19. 接口与页面对照
当前前后端已形成闭环的主要页面包括：
- 游客：首页、套餐列表、套餐详情、中心列表、体检指引、FAQ
- 用户：登录、注册、预约列表、预约创建、预约详情、订单确认、任务引导、报告列表、报告详情、报告对比、咨询医生
- 医生：待检任务、结果录入、报告审核、咨询待办、异常分析、工作量统计
- 运营：套餐管理、排班管理、预约管理、订单管理、退款管理、运营分析
- 管理员：任务生成、报告发布、咨询分配、医生分析

## 20. 当前接口缺口
以下需求在产品文档中存在，但当前接口未形成闭环：
- 用户健康档案维护接口
- 报告下载、分享、导出接口
- 用户管理、角色权限管理、数据字典、系统配置、审计日志管理接口
- 内容运营后台接口（FAQ、体检指南等动态配置）

这些内容应作为后续“可开发需求文档”中的扩展任务处理。
