## 二、第三方 API 配置的 4 种模式

### 模式 A：自定义模型提供者（`config.toml`）

在 `~/.codex/config.toml` 或项目 `.codex/config.toml` 中添加：

```toml
model_provider = "custom"
model = "your-model-name"

[model_providers.custom]
name = "your-provider-name"
wire_api = "chat"              # "chat" = Chat Completions API, "responses" = Responses API
requires_openai_auth = false   # true = 使用 OpenAI 认证, false = 使用自有认证
base_url = "http://your-api-endpoint/v1"
api_key = "your-api-key"       # 可选，也可使用 auth.json
```

**关键字段说明：**
| 字段 | 说明 |
|------|------|
| `name` | 提供者显示名称 |
| `wire_api` | `"chat"`（Chat Completions）或 `"responses"`（Responses API） |
| `requires_openai_auth` | 是否透传 OpenAI 认证 |
| `base_url` | 第三方 API 端点 URL |
| `api_key` | 可选，内联 API 密钥 |

### 模式 B：MCP 服务器（项目级 `.codex/config.toml`）

```toml
[mcp_servers.server-name]
url = "http://host:port/mcp"
```

用于连接外部工具/数据源（如 GitHub、Slack、Notion 等）。

### 模式 C：Shell 环境变量（兼容 Anthropic API）

```toml
[shell_environment_policy.set]
ANTHROPIC_AUTH_TOKEN = "your-token"
ANTHROPIC_BASE_URL = "https://your-anthropic-proxy/anthropic"
ANTHROPIC_DEFAULT_HAIKU_MODEL = "model-name"
ANTHROPIC_DEFAULT_OPUS_MODEL = "model-name"
ANTHROPIC_DEFAULT_SONNET_MODEL = "model-name"
```

### 模式 D：插件级 MCP（通过 `.mcp.json`）

插件可在 `plugin.json` 的 `mcpServers` 字段中捆绑自己的 MCP 服务器配置。

## 三、配置层级（优先级）

Codex 按以下顺序读取配置，后者覆盖前者：
1. `/etc/codex/config.toml`（系统级）
2. `$CODEX_HOME/config.toml`（用户全局，即 `~/.codex/config.toml`）
3. `$CODEX_HOME/<name>.config.toml`（Profile 配置）
4. 项目级 `.codex/config.toml`（受信任项目）

## 四、关键文件路径

| 文件 | 用途 |
|------|------|
| `~/.codex/config.toml` | 全局模型提供者、模型、插件、沙箱配置 |
| `~/.codex/auth.json` | API 密钥认证 |
| 项目 `.codex/config.toml` | 项目级 MCP 服务器配置 |
| 项目 `.mcp.json` | MCP 服务器配置（另一种格式） |
