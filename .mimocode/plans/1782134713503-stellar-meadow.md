## 下单流程问题根因分析

### 状态字段命名混乱

| 位置 | 字段名 | 值 | 问题 |
|---|---|---|---|
| 后端实体 `OrderEntity.status` | `status` | Integer 0/1 | 正确 |
| 新建订单返回 (OrderService:118) | `status` | `"WAIT_PAY"` | 字段名对，值不对 |
| 幂等返回 (OrderService:64) | `orderStatus` | `"PENDING"` | 字段名和值都不对 |
| 前端按钮判断 (OrderConfirm:88) | `orderStatus` | `'PENDING'` | 与新建返回不匹配 |
| 前端状态栏 (OrderConfirm:58) | `orderStatus` | `'PAID'` | 同上 |

### pay() 方法问题

后端 `pay()` 只调用 `paymentService.createPayment()` 创建支付记录，**从未更新订单 status 从 0 到 1**。状态更新只在回调接口 `MockPaymentService.handleCallback()` 中，但前端从未调用回调。

## 问题 8：支付成功后预约状态未更新

### 根因

预约详情页 (`AppointmentDetail.vue`) 显示的是**预约状态**（`appointment.status`），不是订单状态。

- 预约状态：0=待确认, 1=已确认
- 订单状态：0=待支付, 1=已支付

支付成功后，订单状态更新为 1，但预约状态仍为 0（待确认）。

### 修改

**文件**: `backend/.../order/service/OrderService.java` pay() 方法

在更新订单状态后，同时更新对应预约的状态为 1（已确认）：

```java
// 更新订单状态
orderMapper.update(...);

// 更新预约状态为已确认
appointmentMapper.update(null, new LambdaUpdateWrapper<AppointmentEntity>()
        .eq(AppointmentEntity::getId, order.getAppointmentId())
        .eq(AppointmentEntity::getStatus, 0)
        .set(AppointmentEntity::getStatus, 1));
```

需要注入 `AppointmentMapper`。

### 验证

支付成功后，返回预约详情页，状态应显示"已确认"。

## 问题 17：退款状态流转不完整

### 问题描述
1. 申请退款后，订单状态没有更新为"退款中"
2. 退款通过后，预约状态没有更新为"已取消"

### 根因
- `apply` 方法只创建退款申请记录，未更新订单状态
- `approve` 方法只更新订单状态为 2（已退款），未更新预约状态

### 修改

**文件**: `backend/.../order/service/RefundService.java`

#### apply 方法（第 74 行后）
创建退款申请后，更新订单状态为 3（退款中）：
```java
orderMapper.update(null, new LambdaUpdateWrapper<OrderEntity>()
        .eq(OrderEntity::getId, order.getId())
        .set(OrderEntity::getStatus, 3));
```

#### approve 方法（第 154 行后）
更新订单状态后，同时更新预约状态为 3（已取消）：
```java
appointmentMapper.update(null, new LambdaUpdateWrapper<AppointmentEntity>()
        .eq(AppointmentEntity::getId, order.getAppointmentId())
        .set(AppointmentEntity::getStatus, 3));
```

需要注入 `AppointmentMapper`。

### 订单状态定义
- 0: 待支付
- 1: 已支付
- 2: 已退款
- 3: 退款中

### 验证
1. 申请退款后，订单状态变为"退款中"
2. 退款审核通过后，订单状态变为"已退款"，预约状态变为"已取消"

## 问题 18：退款审核报错"当前订单不满足退款条件"

### 根因

`approve` 方法第 125 行检查：
```java
if (order == null || order.getStatus() == null || order.getStatus() != 1)
```

申请退款后，订单状态已从 1（已支付）变为 3（退款中），但审核方法只允许状态 1。

### 修改

**文件**: `backend/.../order/service/RefundService.java` 第 125 行

修改状态检查，允许状态 1（已支付）或 3（退款中）：
```java
if (order == null || order.getStatus() == null || (order.getStatus() != 1 && order.getStatus() != 3))
```

### 关于预约详情页显示"已确认"

这是正确行为。预约状态和订单状态是独立的：
- 申请退款 → 订单变为"退款中"，预约仍为"已确认"
- 审核通过 → 订单变为"已退款"，预约变为"已取消"

只有运营审核通过后，预约状态才会改变。
