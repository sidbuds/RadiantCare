# 熙心健康体检平台需求文档

## 1. 项目背景
熙心健康体检平台面向体检用户、医生/医技录入人员、运营人员、系统管理员，覆盖从健康档案、套餐选择、预约下单、到院导检、分科检查、结果录入、报告生成、报告审核发布、咨询医生、历年对比、健康建议到运营分析的完整闭环。

本平台不是单一“预约系统”或“报告系统”，而是一个围绕体检业务全流程的数据化平台。用户给出的“呼吸道体检 + 血检”只是一个典型例子，系统实际需要支持多套餐、多检查类型、多科室、多角色协作的完整业务链路。

## 2. 项目目标
- 打通“预约 -> 订单 -> 到院 -> 检查 -> 报告 -> 咨询 -> 对比 -> 建议”主链路。
- 支持多角色协同：普通用户、医生/医技录入人员、运营人员、系统管理员。
- 支持多套餐、多科室、多模板、多年度健康沉淀，而非只支持单一套餐示例。
- 让用户获得更清晰的体检流程、更高效的报告交付和更连续的健康管理体验。
- 让医院/体检中心获得更标准的执行流程、更可追溯的数据、更高效的运营管理能力。

## 3. 角色定义
| 角色 | 职责说明 |
|---|---|
| 游客 | 浏览体检套餐、中心介绍、体检流程、注意事项、常见问题，查看部分公开时段信息，并在预约/下单前完成注册或登录 |
| 普通用户 | 维护档案、查询可预约时段、预约体检、创建订单、支付订单、查看导检路线、查看报告、申请退款、咨询医生、查看历年对比与健康建议 |
| 医生/医技录入人员 | 执行检查、上传/录入检查结果、补充结论、审核报告、发布/撤回报告、查看异常数据、处理高风险问题、回复用户咨询 |
| 运营人员 | 管理预约、订单、套餐、排班与时段、退款审核、订单导出、运营分析、趋势分析 |
| 系统管理员 | 用户管理、角色权限、字典配置、基础系统参数维护、审计与安全管理 |

## 5. 平台完整业务链路

### 5.0 游客转化链路
1. 游客进入平台首页，先浏览平台介绍、热门套餐、中心信息和体检流程说明。
2. 游客查看感兴趣套餐的价格、适用人群、检查项、注意事项和常见问题。
3. 游客可查看部分公开时段展示，用于判断某中心近期是否具备预约能力。
4. 当游客点击“立即预约”“立即下单”“咨询医生”等动作时，系统提示先注册或登录。
5. 游客登录后，系统尽量保留已选套餐、中心、日期等上下文，再继续进入正式用户主链路。

### 5.1 用户主链路
1. 用户注册或登录后，先维护个人健康档案、体检人信息、既往史、过敏史等。
2. 用户浏览体检套餐，查看套餐包含的检查项、适用人群、价格、注意事项、可能涉及的科室。
3. 用户选择体检中心、日期、时段，查询可预约时段并创建预约。
4. 用户基于预约创建订单，确认套餐、价格、附加项、优惠信息并完成支付。
5. 到院后，平台根据订单和套餐生成体检任务，并按照套餐项目和中心路线规则输出导检路径。
6. 用户按导检路线依次前往检验科、影像科、内科、妇科等相关科室完成检查。
7. 各科室检查完成后，医生或录入人员在平台录入结果，后端根据套餐模板自动生成结构化报告。
8. 报告审核通过并发布后，用户查看、下载、分享报告，查看异常指标与总体结论。
9. 用户可基于报告、异常项、历年对比结果发起咨询医生。
10. 用户可查看历年报告对比结果、趋势分析和健康建议。

### 5.2 医生/医技执行链路
1. 医生或录入人员登录医生端，查看待处理的体检任务和检查项目。
2. 根据用户体检任务开始检查，确认项目状态从“待检查”进入“检查中”。
3. 将检查结果录入系统，支持数值型、描述型、枚举型、附件型等结果形式。
4. 对需要补充说明的检查项填写结论、建议或备注，并在必要时上传图片/附件。
5. 检查项完成后，任务项状态更新为“已完成”；若需复检或补录，可标记为“待复检/待补录”。
6. 报告生成后，医生/质控人员可执行报告审核，记录审核意见。
7. 审核通过后可发布报告；若发现问题，可撤回报告重新修改。
8. 医生可查看异常数据、进行异常统计、同比环比分析、导出异常数据、处理高风险问题。
9. 医生可在咨询管理中查看用户问题，结合报告和历史趋势进行人工回复。
10. 平台按医生/科室维度统计工作量，用于绩效和排班优化。

### 5.3 运营人员管理链路
1. 运营人员维护体检套餐，配置套餐基础信息、检查项、价格、适用人群、报告模板绑定关系。
2. 运营人员维护预约排班与时段资源，配置中心、日期、时段、容量、可预约规则。
3. 用户预约产生后，运营人员在预约管理中查看预约状态、到检情况、取消情况和爽约情况。
4. 用户下单后，运营人员在订单管理中跟踪待支付、已支付、退款中、已完成等状态。
5. 对用户退款申请进行审核，确认退款条件、退款金额、退款原因和处理结果。
6. 运营人员可导出订单、预约、套餐、趋势等报表，用于日常经营分析。
7. 通过运营数据看板分析预约量、订单量、支付转化、退款率、套餐销售表现。
8. 通过预约趋势分析、订单转化分析、套餐分析持续优化供给策略和活动策略。

### 5.4 系统管理员管理链路
1. 系统管理员维护平台用户、角色、权限和菜单可见范围。
2. 管理基础字典，如检查项目字典、异常等级字典、状态字典、通知模板等。
3. 管理系统参数，如报告分享有效期、预约规则、密码策略、审计保留策略等。
4. 查看关键审计日志，确保报告审核、发布、撤回、退款等敏感操作可追溯。
5. 对平台整体安全、权限边界、系统配置一致性负责。

## 6. 多套餐、多科室、多链路要求
- “呼吸道体检 + 血检”只是一个套餐示例，不是唯一业务链路。
- 系统需支持多种体检链路，包括但不限于：
  - 检验类主导链路：血检、尿检、便检、生化、肿瘤标志物。
  - 影像类主导链路：X 光、CT、MRI、超声。
  - 门诊问诊类链路：内科、外科、耳鼻喉、眼科、口腔、妇科。
  - 性别/年龄差异化链路：女性专项、男性专项、中老年专项、儿童筛查。
- 同一套餐可拆分到多个科室；同一科室也可承接多个检查项。
- 同一用户的一次体检可能存在漏检、补检、复检、撤回重录等复杂场景。
- 同一套餐在不同体检中心可对应不同导检路线和不同资源配置。
- 同一套餐允许配置不同版本的报告模板，以支持后续模板升级。

## 7. 核心功能需求

### 7.1 预约管理
- 支持查询可预约时段、创建预约、取消预约、改约。
- 支持预约容量控制、并发校验、重复预约校验。
- 支持运营侧维护排班与时段资源。

### 7.2 订单管理
- 支持创建订单、支付、查询、导出、退款申请与退款审核。
- 支持订单与预约、套餐、支付记录、退款记录联动。
- 支持复杂订单价格快照、金额分摊和审计留痕。

### 7.3 到院导检与检查执行
- 支持按套餐自动拆分体检任务与任务项。
- 支持导检路线生成、检查项状态跟踪、复检/补检处理。
- 支持医生/医技录入结果、上传附件、填写检查结论。

### 7.4 报告管理
- 支持按套餐模板自动生成结构化报告。
- 支持报告审核、发布、撤回、版本留痕、下载与分享。
- 支持关键指标结构化存储。

### 7.5 异常分析与健康建议
- 支持异常指标标记、异常分级、异常数据统计与导出。
- 支持历年报告对比、趋势判断、健康建议生成。
- 支持高风险问题识别后的人工处理闭环。
- 支持医生查看个人异常总览、异常趋势、高风险用户清单。
- 支持管理员按医生/科室维度查看异常分析汇总。

### 7.6 咨询医生
- 支持用户基于报告、异常项、历年对比结果发起咨询医生。
- 支持医生查看问题、引用报告和指标进行人工回复。
- 支持咨询状态跟踪、回复记录留痕和处理时间记录。
- 首期不做 AI 自动问答，仅做人工作答复闭环。

## 8. 非功能要求
- 高并发预约场景下，关键接口需具备稳定性和限流能力。
- 报告、检查结果、咨询记录、审核记录需可追溯。
- 敏感个人信息需加密存储与传输。
- 医生录入、报告审核、退款审核、权限变更必须留痕。
- 模块设计需支持后续扩展到团检、企业客户、多中心管理。

## 9. 验收标准
- 游客链路可完整跑通：浏览 -> 查看详情 -> 查看说明 -> 登录转化。
- 用户主链路可完整跑通：档案 -> 预约 -> 订单 -> 支付 -> 到院 -> 导检 -> 检查 -> 报告 -> 咨询 -> 对比 -> 建议。
- 医生链路可完整跑通：接诊/检查 -> 录入结果 -> 审核/发布报告 -> 回复咨询 -> 异常分析。
- 运营链路可完整跑通：套餐配置 -> 排班时段 -> 预约管理 -> 订单管理 -> 退款审核 -> 数据分析。
- 管理链路可完整跑通：用户管理 -> 权限配置 -> 系统配置 -> 审计追踪。
- 所有关键状态流转都能在数据库中落库并追溯。

## 10. 当前第一阶段已落地能力
- 认证已从硬编码登录切换为数据库登录。
- 用户端已支持预约、订单、模拟支付、退款申请、报告查看、咨询医生、双报告对比、健康建议。
- 医生端已支持体检任务处理、结果录入、咨询回复、异常分析、工作量统计。
- 运营端已支持独立 `OPERATOR` 角色登录，以及套餐管理、排班管理、预约管理、订单管理、退款审核、运营看板。
- 管理端当前保留体检任务生成、报告生成审核发布、咨询分配、医生分析总览等系统级入口。
- 游客公开浏览链路已在文档中补充设计，但后端公开接口当前尚未实现。
