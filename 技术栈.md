# 技术栈

## 1. 文档目标
本文档用于统一说明熙心健康体检平台当前使用的技术栈，替代仓库中分散的前端/后端技术说明文档。

## 2. 总体架构
- 前端：单页应用（SPA）
- 后端：单体 Spring Boot 服务
- 数据库：MySQL
- 缓存：Redis
- 部署方式：本地分进程启动 或 Docker Compose 一键启动

## 3. 前端技术栈
### 3.1 核心框架
- Vue 3
- TypeScript
- Vite
- Vue Router
- Pinia

### 3.2 UI 与可视化
- Element Plus
- Element Plus Icons
- ECharts
- Sass

### 3.3 工程与质量
- Vitest：单元测试
- Vue Test Utils：组件测试
- vue-tsc：类型检查

### 3.4 当前前端特征
- 已实现角色路由与登录权限守卫
- 已实现游客、用户、医生、运营、管理员页面
- 已实现请求统一封装
- 已支持亮暗主题系统

## 4. 后端技术栈
### 4.1 核心框架
- Java 8
- Spring Boot 2.7.18
- Spring Security
- JWT

### 4.2 数据层
- MySQL 8
- MyBatis-Plus
- PageHelper

### 4.3 文档与工具
- SpringDoc OpenAPI（Swagger UI）
- Lombok
- Spring Boot Test

### 4.4 当前后端特征
- 已实现统一返回结构 `ApiResult`
- 已实现 JWT 登录与角色校验
- 已实现预约、订单、任务、报告、咨询、对比、运营、管理等业务模块
- 已实现公开接口、用户接口、医生接口、运营接口、管理员接口分层

## 5. 基础设施
### 5.1 数据库与缓存
- MySQL 8
- Redis 7

### 5.2 容器化
- Docker
- Docker Compose
- Nginx（前端容器）

### 5.3 当前容器化能力
- 已支持 MySQL、Redis、后端、前端四服务编排
- 已支持数据库初始化脚本自动导入
- 已支持环境变量统一管理
- 已支持后端重新构建并重启

## 6. 开发与运行方式
### 6.1 本地开发
- 前端：`npm run dev`
- 后端：`mvn spring-boot:run`
- 数据库依赖：Docker 中的 MySQL / Redis

### 6.2 容器化开发
- 通过 `docker-compose.yml` 启动全部服务
- 通过 `docker-up.ps1` 执行构建与启动
- 前端端口：`5173`
- 后端端口：`18080`
- MySQL 端口：`3307`（宿主） / `3306`（容器内）
- Redis 端口：`6379`

## 7. 技术栈优势
- 架构清晰，前端 SPA + 后端单体服务，团队上手快。
- 前端 Element Plus 适合中后台体检业务开发。
- 后端 MyBatis-Plus 提高 CRUD 开发效率。
- Docker Compose 能快速拉起完整开发环境。
- 数据库结构与业务模块已覆盖体检平台主链路。

## 8. 技术栈限制与建议
### 8.1 当前限制
- Java 8 与 Spring Boot 2.7 属于老版本组合，后续升级空间有限。
- 当前为单体服务，若业务规模增长，需要关注后续拆分策略。
- 内容管理、权限管理、审计管理模块当前仍缺失完整后台实现。

### 8.2 后续建议
- 若团队允许，可规划从 Spring Boot 2.7 向 Spring Boot 3.x 渐进升级。
- 若并发提升明显，可进一步完善 Redis 缓存与限流策略。
- 若后台功能继续扩展，建议补齐 RBAC + 字典 + 配置 + 审计模块。
