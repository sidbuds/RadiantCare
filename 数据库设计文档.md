# 熙心健康体检平台数据库设计文档

## 1. 设计原则
- 主键统一使用 `BIGINT AUTO_INCREMENT`
- 业务单号统一独立字段并加唯一索引
- 时间字段统一使用 `DATETIME(3)`
- 审计字段统一包含 `created_at / updated_at / created_by / updated_by / is_deleted`
- 当前阶段以逻辑关联为主，不强制物理外键

## 4. 关键设计说明

### 4.1 为什么后台账号不直接塞进 `user`
- 用户端和后台端职责不同
- 医生、运营、管理员属于后台身份
- 独立 `staff_account` 更利于权限、审计和后续多角色扩展

### 4.2 为什么 `user` 仍补 `password_hash`
- 当前登录入口统一为 `/api/auth/login`
- 第一阶段要把普通用户也切到数据库认证
- 用 `user_no + password_hash` 可以最小改动完成兼容

### 4.3 为什么 `resource_capacity` 要补 `status`
- 运营端需要启停时段
- 仅靠 `capacity_total = 0` 无法表达“停用但保留历史配置”

### 4.4 为什么医生分析首期不加统计表
- 当前数据量下实时聚合更直观
- 现有表已能支撑医生个人分析和管理员汇总分析
- 先把统计口径固定，再决定是否引入日汇总表

## 5. 医生异常分析与工作量统计依赖表

### 5.1 异常分析
- `exam_result`
  - `entry_doctor_id`
  - `entry_time`
  - `is_abnormal`
  - `abnormal_level`
  - `task_item_id`
- `exam_task_item`
  - `department_code`
  - `department_name`
- `user`
  - 用于高风险用户清单展示姓名、手机号

### 5.2 工作量统计
- `exam_task_item`
  - 统计任务项总量、完成量、平均处理时长
- `exam_result`
  - 统计结果录入量
- `doctor_review_record`
  - 统计报告审核量
- `doctor_consultation_reply`
  - 统计医生回复咨询量

## 6. 当前数据库落地边界
- 已支持数据库登录与角色扩展
- 已支持运营端第一阶段管理与分析
- 已支持医生异常分析与工作量统计第一阶段，无需额外 DDL
- 游客公开浏览链路当前仅停留在文档设计层，尚无对应公开接口落地
- 暂未实现完整 RBAC 菜单权限表
- 暂未实现独立导出文件表
- 暂未实现医生分析离线汇总表
